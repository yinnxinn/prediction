## 2. 产品目标与价值

### 2.1 目标

构建一个「电价趋势预测 + 成本预警 + 用电策略建议」的轻量应用，帮助用户提前判断后续电价走势，优化采购和用电计划。

### 2.2 目标用户

- 工商业用电管理人员
- 园区/工厂能源管理人员
- 需要做月度/季度电费预算的财务与运营角色

### 2.3 核心价值

- 从“事后核算”转向“事前预测”
- 给出未来 1-7 天（短期）和未来 1-3 个月（中期）的电价趋势与区间
- 对异常波动给出可解释原因（季节、时段、政策、负荷）

## 3. MVP 功能设计（第一版）

### 3.1 数据看板

- 历史电价趋势（按日/小时）
- 峰平谷电价分布
- 历史均值、波动率、环比/同比

### 3.2 预测中心

- 预测未来 N 天电价（默认 7 天）
- 输出点预测 + 置信区间（上限/下限）
- 支持按时段（峰/平/谷）分别预测

### 3.3 告警与建议

- 当预测电价超过阈值时预警
- 给出简单策略建议（如“高价时段降负荷”“可转移负荷至谷段”）

### 3.4 数据管理

- 导入历史数据（CSV/Excel）
- 字段映射与校验（时间、电价、电量、时段类型）
- 缺失值与异常值处理日志

## 4. 数据模型与口径设计

## 4.1 建议最小数据字段（必须）

- `timestamp`：时间戳（小时级优先）
- `price`：该时点电价（元/kWh）
- `period_type`：峰/平/谷（可选，但强烈建议）
- `consumption`：用电量（kWh，可选）

### 4.2 推荐增强字段（提升精度）

- `region`：地区
- `temperature`：气温
- `holiday_flag`：节假日标记
- `weekday`：星期几
- `policy_tag`：政策变更标记

### 4.3 标签与训练样本构建

- 目标标签：`price(t+h)`，h 为预测步长
- 滑窗特征：过去 24/72/168 小时的价格统计特征
- 时序特征：小时、星期、月份、节假日
- 分组训练：按峰/平/谷分模型，或用同一模型带分类特征

## 5. 技术实现路径（分阶段）

### 阶段 A：可用版本（2-3 周）

- 目标：快速得到可展示、可评估的预测结果
- 模型：`LightGBM/XGBoost` 回归（稳定、训练快、可解释）
- 预测：滚动预测未来 7 天
- 评估：`MAE`、`RMSE`、`MAPE`
- 交付：基础 Web 看板 + 预测 API + 导入功能

### 阶段 B：增强版本（3-6 周）

- 增加外部变量：天气、节假日、政策事件
- 尝试时序深度模型：`LSTM` / `Temporal Fusion Transformer`
- 引入分位数预测（P10/P50/P90）增强风险表达
- 增加异常检测模块（突变点识别）

### 阶段 C：运营版本（持续迭代）

- 自动重训练（每周/每月）
- 模型监控（漂移监控、误差告警）
- 建议引擎（结合负荷转移规则生成可执行建议）

## 6. 建议技术栈

- 前端：`React + Ant Design + ECharts`
- 后端：`FastAPI`（Python）
- 训练与推理：`pandas + scikit-learn + lightgbm`
- 任务调度：`APScheduler` 或 `Airflow`（后续）
- 存储：`PostgreSQL`（业务数据）+ `MinIO`（模型文件，可选）

## 7. 系统架构（简版）

1. 数据导入层：上传 CSV/Excel -> 校验 -> 入库  
2. 特征工程层：时间特征、统计特征、缺失处理  
3. 模型层：训练、评估、版本管理  
4. 服务层：预测 API、告警 API、报表 API  
5. 展示层：看板、趋势、预警、建议  

## 8. 成功标准（MVP）

- 关键场景可用：可完成导入、训练、预测、展示全链路
- 预测精度达到基线：MAPE <= 10%-15%（按场景可调整）
- 能解释结果：支持特征重要性与分时段误差分析
